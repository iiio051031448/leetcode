## 题目描述

```
给定一个数组，它的第 i 个元素是一支给定股票第 i 天的价格。

如果你最多只允许完成一笔交易（即买入和卖出一支股票一次），设计一个算法来计算你所能获取的最大利润。

注意：你不能在买入股票前卖出股票。

 

示例 1:

输入: [7,1,5,3,6,4]
输出: 5
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。
     注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。
示例 2:

输入: [7,6,4,3,1]
输出: 0
解释: 在这种情况下, 没有交易完成, 所以最大利润为 0。
```

## 思路

由于我们是想获取到最大的利润，我们的策略应该是低点买入，高点卖出。

由于题目对于交易次数有限制，只能交易一次，因此问题的本质其实就是求波峰浪谷的差值的最大值。

## 代码

语言支持：JS，C++，Java，Python

JS Code:

```js
/**
 * @param {number[]} prices
 * @return {number}
 */
var maxProfit = function (prices) {
  let min = prices[0];
  let profit = 0;
  // 7 1 5 3 6 4
  for (let i = 1; i < prices.length; i++) {
    if (prices[i] > prices[i - 1]) {
      profit = Math.max(profit, prices[i] - min);
    } else {
      min = Math.min(min, prices[i]);
    }
  }

  return profit;
};
```

LYJ C++ : 如下代码完全可行，而且少了一个判断 "if (prices[i] > prices[i -1])"，性能更高。
```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        if (prices.empty()) return 0;
        auto min = prices[0];
        auto profit = 0;
        for (auto i = 1; i < prices.size(); i++) {
            profit = max(profit, prices[i] - min);
            min = std::min(min, prices[i]);
        }
        return profit;
    }
};
```

### LYJ: 上面两种写法比较：


第二种写法不仅能通过，而且在实际开发中往往更受欢迎。这两种写法在逻辑上都是完全正确的，它们之间的区别主要在于**逻辑分支优化**与**代码简洁性**的权衡。

下面为您分析一下为什么第二种化简后的写法同样正确，以及它们各自的特点：

#### 为什么第二种写法的逻辑是安全的？

在第一种写法中，您通过 `if (prices[i] > prices[i - 1])` 进行了分流：
1. **价格上涨时**（`prices[i] > prices[i - 1]`）：此时有可能产生更大的利润，因此更新 `profit`。由于价格在上涨，当前的 `prices[i]` 肯定不是历史最低点（至少比前一天高），所以此时不需要更新 `min`。
2. **价格下跌或持平时**（`prices[i] <= prices[i - 1]`）：此时不可能产生新的最大利润（因为价格比前一天低，今天卖出一定不如之前卖出划算），但有可能诞生新的历史最低价，因此只需更新 `min`。

而在第二种写法中，去掉了这个 `if-else` 分支，改为**无条件更新**这两项。由于 `std::max` 和 `std::min` 的数学性质，这种无条件更新在逻辑上依然是安全的：
* **在下跌日**：计算出的临时利润 `prices[i] - min` 一定小于或等于之前的利润，因此 `max(profit, ...)` 运行后，`profit` 保持不变，不会被错误地改小。
* **在上涨日**：当前的 `prices[i]` 较大，因此 `std::min(min, prices[i])` 运行后，`min` 依然会保持之前的历史最低值，不会被错误地改大。

---

#### 两种写法的对比

| 维度 | 第一种写法（带 `if-else`） | 第二种写法（无条件更新） |
| :--- | :--- | :--- |
| **可读性** | 逻辑稍显繁琐，需要理解分支的意图。 | **更加简洁直观**，核心逻辑只有两行，不易写出 Bug。 |
| **执行开销** | 减少了部分 `max` 或 `min` 的比较和赋值操作。 | 每次循环都需要执行 `max` 和 `min`。 |
| **CPU 分支预测** | 引入了条件分支。如果股票价格频繁波动，可能会产生**分支预测失败**的开销。 | **无分支（Branchless）**。现代编译器通常会将无分支的 `std::min/max` 优化为 CPU 的条件传送指令（如 `cmov`），在某些高频波动的输入下，运行速度反而可能更快。 |



Java Code:

```java
class Solution {
    public int maxProfit(int[] prices) {
        int minprice = Integer.MAX_VALUE;
        int maxprofit = 0;
        for (int price: prices) {
            maxprofit = Math.max(maxprofit, price - minprice);
             minprice = Math.min(price, minprice);
        }
        return maxprofit;
    }
}
```

Python Code:

```python
class Solution:
    def maxProfit(self, prices: 'List[int]') -> int:
        if not prices: return 0

        min_price = float('inf')
        max_profit = 0

        for price in prices:
            if price < min_price:
                min_price = price
            elif max_profit < price - min_price:
                max_profit = price - min_price
        return max_profit
```
