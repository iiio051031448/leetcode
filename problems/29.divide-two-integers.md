## 题目描述

```
给定两个整数，被除数 dividend 和除数 divisor。将两数相除，要求不使用乘法、除法和 mod 运算符。

返回被除数 dividend 除以除数 divisor 得到的商。

整数除法的结果应当截去（truncate）其小数部分，例如：truncate(8.345) = 8 以及 truncate(-2.7335) = -2

 

示例 1:

输入: dividend = 10, divisor = 3
输出: 3
解释: 10/3 = truncate(3.33333..) = truncate(3) = 3
示例 2:

输入: dividend = 7, divisor = -3
输出: -2
解释: 7/-3 = truncate(-2.33333..) = -2
 

提示：

被除数和除数均为 32 位有符号整数。
除数不为 0。
假设我们的环境只能存储 32 位有符号整数，其数值范围是 [−231,  231 − 1]。本题中，如果除法结果溢出，则返回 231 − 1。

```

## 思路

符合直觉的做法是，减数一次一次减去被减数，不断更新差，直到差小于 0，我们减了多少次，结果就是多少。

核心代码：

```js
let acc = divisor;
let count = 0;

while (dividend - acc >= 0) {
  acc += divisor;
  count++;
}

return count;
```

这种做法简单直观，但是性能却比较差. 下面来介绍一种性能更好的方法。

const isNegative = dividend > 0 !== divisor > 0;

const isNegative = dividend ^ (divisor < 0);

## 代码

- 语言支持：JS, Python3, CPP

```js
/*
 * @lc app=leetcode id=29 lang=javascript
 *
 * [29] Divide Two Integers
 */
/**
 * @param {number} dividend
 * @param {number} divisor
 * @return {number}
 */
var divide = function (dividend, divisor) {
  if (divisor === 1) return dividend;

  // 这种方法很巧妙，即符号相同则为正，不同则为负
  const isNegative = dividend > 0 !== divisor > 0;

  const MAX_INTERGER = Math.pow(2, 31);

  const res = helper(Math.abs(dividend), Math.abs(divisor));

  // overflow
  if (res > MAX_INTERGER - 1 || res < -1 * MAX_INTERGER) {
    return MAX_INTERGER - 1;
  }

  return isNegative ? -1 * res : res;
};

function helper(dividend, divisor) {
  // 二分法
  if (dividend <= 0) return 0;
  if (dividend < divisor) return 0;
  if (divisor === 1) return dividend;

  let acc = 2 * divisor;
  let count = 1;

  while (dividend - acc > 0) {
    acc += acc;
    count += count;
  }
  // 直接使用位移运算，比如acc >> 1会有问题
  const last = dividend - Math.floor(acc / 2);

  return count + helper(last, divisor);
}
```

Python3 Code:

```python
class Solution:
    def divide(self, dividend: int, divisor: int) -> int:
        """
        二分法
        :param int divisor
        :param int dividend
        :return int
        """
        # 错误处理
        if divisor == 0:
            raise ZeroDivisionError
        if abs(divisor) == 1:
            result = dividend if 1 == divisor else -dividend
            return min(2**31-1, max(-2**31, result))

        # 确定结果的符号
        sign = ((dividend >= 0) == (divisor >= 0))

        result = 0
        # abs也可以直接写在while条件中，不过可能会多计算几次
        _divisor = abs(divisor)
        _dividend = abs(dividend)

        while _divisor <= _dividend:
            r, _dividend = self._multi_divide(_divisor, _dividend)
            result += r

        result = result if sign else -result

        # 注意返回值不能超过32位有符号数的表示范围
        return min(2**31-1, max(-2**31, result))

    def _multi_divide(self, divisor, dividend):
        """
        翻倍除法，如果可以被除，则下一步除数翻倍，直至除数大于被除数，
        返回商加总的结果与被除数的剩余值；
        这里就不做异常处理了；
        :param int divisor
        :param int dividend
        :return tuple result, left_dividend
        """
        result = 0
        times_count = 1
        while divisor <= dividend:
            dividend -= divisor
            result += times_count
            times_count += times_count
            divisor += divisor
        return result, dividend
```

**代码中的潜在越界与 Bug 分析**

虽然这个解法非常巧妙，但在处理 LeetCode 的特定边界测试用例时，存在一个**未定义行为（UB）或 Bug**。

#### 1. 溢出边界：`INT_MIN / -1`
当输入 `dividend = INT_MIN`（$-2147483648$）且 `divisor = -1` 时：
* 该除法的理论结果是 $2147483648$，这超出了 32 位有符号正数的最大值 `INT_MAX`（$2147483647$）。根据 LeetCode 要求，此时应当返回 `INT_MAX`。
* 在该代码中：
  * 循环结束时，计算出的商为 `q = INT_MIN`（$-2147483648$）。
  * 符号 `pos` 为 `true`，代码执行 `return -q;`。
  * 在 C++ 中，`-INT_MIN` 会导致有符号整型溢出。因为 $-(-2147483648)$ 无法用 32 位 `int` 正数表示，溢出后的值通常仍为 `-2147483648`（即 `INT_MIN`），这会导致结果错误，并且在现代编译器中属于**未定义行为**。

#### 2. 如何修复？
为了使代码完全通过 LeetCode 并避免溢出错误，可以在函数开头添加一行针对该特殊溢出情况的特判：

```cpp
if (dividend == INT_MIN && divisor == -1) {
    return INT_MAX; // 特判唯一会溢出到 INT_MAX 之外的边界情况
}
```

```cpp
class Solution {
public:
    int divide(int dividend, int divisor) {
        // 1. 除数为 0 的安全检查
        if (!divisor) return 0;

        if (dividend == INT_MIN && divisor == -1) {
            return INT_MAX; // 特判唯一会溢出到 INT_MAX 之外的边界情况
        }

        // 2. 确定最终结果的符号
        // pos1 和 pos2 分别表示被除数和除数是否为正数。
        // 通过异或运算 ^ 判断两者符号是否相同：
        // 若相同，则 pos 为 true（结果为正）；若不同，则 pos 为 false（结果为负）。
        bool pos1 = dividend > 0, pos2 = divisor > 0, pos = !(pos1^pos2);

        // 3. 统一转换为负数进行处理
        if (pos1) dividend = -dividend;
        if (pos2) divisor = -divisor;

        // 4. 初始化变量
        int q = 0;             // 累加的商（用负数表示）
        int d = divisor;       // 当前翻倍后的除数
        int t = 1;             // 当前翻倍后的商的步长（对应的倍数）

        // 5. 核心循环：当步长 t 还大于 0，且被除数尚未被减完（因为是负数，所以是 < 0）
        while (t > 0 && dividend < 0) {
            // 此时 dividend 和 d 都是负数
            // dividend - d <= 0 等价于 |dividend| >= |d|，说明还可以继续减
            if (dividend - d <= 0) {
                dividend -= d; // 减去当前翻倍后的除数
                q -= t;        // 累加对应的商（因为 q 是负数表示，所以用 -=）

                // 尝试将除数和步长翻倍，以加速下一次相减
                // 安全检查：(INT_MIN >> 1) < d 确保 d 翻倍后不会超过 INT_MIN 溢出
                if ((INT_MIN >> 1) < d) {
                    t <<= 1;
                    d <<= 1;
                }
            } else {
                // 如果 dividend - d > 0，说明当前翻倍后的 d 太大了，无法再减
                // 此时将除数和步长折半，退回上一个较小的状态
                d >>= 1;
                t >>= 1;
            }
        }

        // 6. 根据最初确定的符号返回最终结果
        return pos? -q : q;
    }
};
```

原CPP 代码：

```cpp
class Solution {
public:
    int divide(int dividend, int divisor) {
        if (!divisor) return 0;  // divide-by-zero error
        bool pos1 = dividend > 0, pos2 = divisor > 0, pos = !(pos1^pos2);
        if (pos1) dividend = -dividend;
        if (pos2) divisor = -divisor;
        int q = 0, d = divisor, t = 1;
        while (t > 0 && dividend < 0) {
            if (dividend - d <= 0) {
                dividend -= d;
                q -= t;
                if ((INT_MIN >> 1) < d) {
                    t <<= 1;
                    d <<= 1;
                }
            } else {
                d >>= 1;
                t >>= 1;
            }
        }
        return pos? -q : q;
    }
};
```
